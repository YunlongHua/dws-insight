# DWS 客户端应用设计

## 1. 项目概述

**项目名称:** DWS Client
**项目类型:** 桌面客户端（Electron）
**核心功能:** 连接 DWS 集群进行 SQL 调优和测试报告生成
**目标用户:** DWS 开发/运维人员 + 测试团队

---

## 2. 技术栈

| 类别 | 技术选型 |
|------|----------|
| 框架 | Electron + React + TypeScript |
| 数据库直连 | pg (node-postgres) |
| 本地配置 | electron-store（加密敏感信息） |
| 业务数据 | better-sqlite3 |
| 大模型接口 | 统一 Gateway（OpenAI/EI-MaaS/自建） |
| 报告导出 | docx + markdown-pdf |
| UI 风格 | 现代简约 / 蓝色科技风 |

---

## 4. 功能模块

### 4.1 ClusterManager — 集群管理
- 多集群配置（测试/预生产/生产）
- 连接测试
- 集群切换
- 敏感信息加密存储（密码、认证信息）

### 4.2 LLMGateway — 大模型网关
- OpenAI 兼容 API
- 华为云 EI-MaaS
- 自建模型
- 模型配置管理

### 4.3 ChatEngine — 对话引擎
- 完整对话上下文管理
- 多轮对话支持
- 对话历史存储

### 4.4 DWSDirector — SQL 执行器
- SQL 执行（DDL/DML/DQL）
- 执行计划获取
- 结果集展示
- 执行时间统计

### 4.5 TuningAdvisor — 调优分析器
- 规则引擎初步分析
- 大模型深度分析
- 生成优化建议
- 用户确认后执行优化
- **优化前后结果对比**

### 4.6 TestRunner — 测试执行器
- 自然语言需求解析
- 测试用例表格自动填充（大模型）
- 用户确认
- 步骤执行与结果写入
- 规则 + 大模型双重判断
- 报错自动纠错与重试

### 4.7 ReportExporter — 报告导出
- 测试用例表格（名称、前提条件、步骤、预期结果、测试结果、备注）
- 可展开详情（完整执行输出）
- 导出格式：Word / Markdown / PDF

---

## 5. 工作流

### 5.1 工作流 A：DWS 调优

```
用户输入 SQL / 调优需求
       ↓
DWSDirector 执行，获取原始结果和执行计划
       ↓
TuningAdvisor（规则 + 大模型）分析
       ↓
展示优化方案 → 用户确认
       ↓
执行优化后的 SQL
       ↓
对比原始结果 vs 优化后结果
       ↓
展示对比报告（执行时间、计划差异、结果一致性）
```

### 5.2 工作流 B：测试报告生成

```
用户输入自然语言需求
       ↓
大模型解析 → 填充测试用例表格
       ↓
用户确认 → TestRunner 执行
       ↓
每步执行结果写入表格对应行
       ↓
规则判断结果 + 大模型评估
       ↓
成功 → 下一条 / 失败 → 大模型纠错 → 重试
       ↓
ReportExporter 生成报告（Word/Markdown/PDF）
```

## 7. 安全

- 敏感信息（密码、API Key）使用 electron-store 加密存储
- 数据库连接信息不上传，仅本地使用
